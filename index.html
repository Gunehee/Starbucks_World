What it does

Built for real-world exploration

Every feature was designed to help users quickly find and learn about a Starbucks anywhere in the world — even with tens of thousands of locations on the map.

🌍

Worldwide Coverage

Every country with a mapped Starbucks presence in OpenStreetMap, from the US to South Korea to the UAE — one map, clustered for performance.

🔍

Search & Country Filter

Type a store name, city, or address, or narrow down to a single country from the dropdown — the map and list update instantly.

☕

Per-Country Menu Reference

Each store's detail panel shows a representative menu and price list for its country — a helpful reference, not live per-store pricing.

📍

Store Detail Panel

Selecting a store opens a slide-in panel with its full address, phone number, hours of operation, and store webpage link where available.

🔄

Automatic Daily Refresh

A GitHub Actions workflow re-queries OpenStreetMap every day, so new and closed stores make their way onto the map without manual work.

↗️

Google Maps Directions

Every store detail panel includes a one-click "Get Directions" button that opens Google Maps with the destination pre-filled.

Data & Methodology

From OpenStreetMap to interactive GeoJSON

Store locations come from OpenStreetMap, queried live through the Overpass API. The project originally used a static 2021 Kaggle CSV filtered to Seattle only — that snapshot is retired in favor of a source that can be re-queried and kept current.

1

Query — For each of ~80 countries, ask Overpass for every node tagged brand:wikidata=Q37158 (Starbucks) inside that country's boundary

2

Dedupe & Normalize — Merge results by OSM node ID, extract name, address, phone, hours, and website from OSM tags

3

Safety Check — If a run returns far fewer stores than the last one (likely a partial Overpass outage), keep the previous good data instead of overwriting it

4

Publish — Save one worldwide GeoJSON file, fetched by the map at runtime and paired with a manually curated per-country menu reference

FileDescription
starbucks_world.geojsonAll worldwide store locations, auto-refreshed daily
menu.jsonReference menu + representative pricing per country

Starbucks has no public per-store pricing API, so menu prices are manually curated country-level references — not live, store-specific prices. Store location data is fetched by map.html at runtime, so viewing the map requires either GitHub Pages or a local static server.

Built with

Open-source, zero dependency

No build tools. No framework. No API key. The map runs from static files — HTML, CSS, and vanilla JS — served by GitHub Pages.

🗺️

Leaflet.js v1.9.4

Interactive map rendering, marker management, and popup system

🔗

Leaflet.markercluster

Clusters tens of thousands of markers so the map stays smooth at world zoom

🌐

OpenStreetMap + Overpass

Free base map tiles and a live, queryable worldwide store dataset — no API key

📦

GeoJSON

Store location data fetched at runtime, with name, address, phone, and hours

⚡

Vanilla JS ES6+

All filtering, search, and state management — no framework dependency

🐍

Python + GitHub Actions

A scheduled workflow re-queries Overpass daily and commits the refreshed data
